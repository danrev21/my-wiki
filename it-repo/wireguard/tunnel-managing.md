
Управлять WireGuard через `systemctl` и `wg-quick` — это стандартный и самый удобный способ на любом современном Linux-дистрибутиве. `wg-quick` отвечает за настройку интерфейса, а `systemctl` позволяет управлять этим как полноценной системной службой.

### Шаг 1: Подготовка конфигурации

Скопируйте ваш конфигурационный файл (допустим, он называется `myvpn.conf`) в директорию `/etc/wireguard/`.

Обратите внимание на имя файла, так как оно будет использоваться для всех команд:
```bash
sudo cp ~/Downloads/myvpn.conf /etc/wireguard/wg0.conf
```

> **Важно:** `systemctl` работает с файлами через шаблон `wg-quick@<имя_интерфейса>`. Если вы назовете файл `wg0.conf`, имя службы будет `wg-quick@wg0`. Если назовете `us.conf` — служба `wg-quick@us` и так далее.

### Шаг 2: Управление туннелем (базовые команды)

Самые частые операции выполняются через стандартные команды `systemctl`:

*   **Запуск туннеля сейчас:**
    `sudo systemctl start wg-quick@wg0`

*   **Остановка туннеля:**
    `sudo systemctl stop wg-quick@wg0`

*   **Перезапуск туннеля (нужен после правки конфига):**
    `sudo systemctl restart wg-quick@wg0`

*   **Проверка статуса (полезно для отладки):**
    `sudo systemctl status wg-quick@wg0`

*   **Посмотреть логи (если что-то пошло не так):**
    `journalctl -u wg-quick@wg0 -n 50`

### Шаг 3: Автоматический запуск (Автозагрузка)

Чтобы VPN поднимался сам при включении компьютера, добавьте службу в автозагрузку:
```bash
sudo systemctl enable wg-quick@wg0
```

Теперь туннель будет подниматься каждый раз при старте системы и автоматически переподниматься при сбоях.

### Сравнение: systemctl vs. wg-quick

Оба инструмента делают одно и то же, но подход отличается:

*   **`wg-quick up wg0`** (прямой запуск): Процесс запуска привязан к вашей терминальной сессии. Если вы закроете терминал или выйдете из системы, туннель **останется включенным**, но управлять им из другого окна будет не так удобно.
*   **`systemctl start wg-quick@wg0`** (сервисная модель): Сервис работает в фоне под надзором systemd. Это позволяет:
    *   Ставить автозагрузку (`enable`).
    *   Ловить ошибки через `journalctl`.
    *   Использовать привилегии `sudo` без лишних предупреждений о правах на конфиг.

### Типичные проблемы и решения

1.  **Ошибка `wg0 already exists`**
    Чаще всего возникает, если интерфейс уже запущен вручную.
    **Решение:** Сначала убейте интерфейс вручную, потом запускайте через сервис:
    ```bash
    sudo wg-quick down wg0
    sudo systemctl start wg-quick@wg0
    ```

2.  **Ошибка `Failed to set DNS configuration`**
    Это происходит, когда `wg-quick` пытается обновить DNS через `resolvconf`, но не находит `systemd-resolved`.
    **Решение:** Установите и включите `systemd-resolved`:
    ```bash
    sudo apt install systemd-resolved     # Для Debian/Ubuntu
    sudo dnf install systemd-resolved     # Для Fedora/CentOS
    sudo systemctl enable --now systemd-resolved
    ```
    Либо удалите строку `DNS = ...` из конфига, если она не нужна.

3.  **Как перезагрузить конфиг без разрыва соединения**
    Хотя `restart` выключает интерфейс на секунду, `wg-quick` поддерживает мягкую перезагрузку:
    ```bash
    sudo systemctl reload wg-quick@wg0
    ```
    Эта команда применит изменения в списке Peer'ов, не трогая сам интерфейс (туннель не "упадет").

### Резюме команд

| Действие | Команда |
| :--- | :--- |
| **Запустить** | `sudo systemctl start wg-quick@wg0` |
| **Остановить** | `sudo systemctl stop wg-quick@wg0` |
| **Перезапустить** | `sudo systemctl restart wg-quick@wg0` |
| **Мягкая перезагрузка (reload)** | `sudo systemctl reload wg-quick@wg0` |
| **Автозагрузка** | `sudo systemctl enable wg-quick@wg0` |
| **Проверить статус** | `systemctl status wg-quick@wg0` |
| **Посмотреть логи** | `journalctl -u wg-quick@wg0 -f` |
