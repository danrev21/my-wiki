Полный процесс переноса БД Zabbix на отдельный сервер

Памятка: что нельзя делать при переносе
❌ Не делайте	                                ✅ Вместо этого
Не останавливая Zabbix Server, делать дамп	Сначала systemctl stop zabbix-server
Дамп без --routines и --triggers		Всегда используйте эти ключи
Копировать /var/lib/mysql целиком       	Используйте mysqldump для совместимости
Менять конфиг Zabbix до проверки сети    	Сначала добейтесь nc -zv SUCCESS
Забыть про веб-конфиг				Править оба файла: zabbix_server.conf и zabbix.conf.php
Удалять старую БД сразу				Подождите несколько дней, убедитесь в стабильности

-----
Исходные данные вашей инфраструктуры:
Компонент	Значение
Hypervisor	Proxmox
Исходный сервер	Zabbix Server + БД на одной ВМ
Новый БД-сервер	192.168.0.30
Zabbix-сервер	192.168.0.90
СУБД	MySQL / MariaDB
Порт БД	3306

Этап 1: Подготовка нового сервера БД
1.1 Создание ВМ в Proxmox
Создайте новую ВМ или LXC-контейнер в Proxmox:

ОС: используйте ту же ОС, что на основном сервере Zabbix (Ubuntu/Debian/CentOS)

Ресурсы: минимум 4GB RAM + быстрый диск (SSD/NVMe)

Сеть: убедитесь, что ВМ в одной сети с Zabbix-сервером

1.2 Установка СУБД
На новом сервере (192.168.0.30) установите MySQL/MariaDB:

bash
# Ubuntu/Debian (MariaDB)
sudo apt update
sudo apt install mariadb-server mariadb-client

# Или MySQL
sudo apt install mysql-server
1.3 Базовая настройка безопасности БД
bash
sudo mysql_secure_installation
Установите root-пароль
Отключите анонимных пользователей
Запретите удалённый root-доступ
Удалите тестовые БД

Этап 2: Настройка сети и доступа к БД

2.1 Разрешаем удалённые подключения к MySQL
На новом БД-сервере отредактируйте конфигурационный файл:
# Для MariaDB
sudo nano /etc/mysql/mariadb.conf.d/50-server.cnf
# Для MySQL
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
Найдите и измените строку:
bind-address = 127.0.0.1    # было
bind-address = 0.0.0.0      # стало — слушаем все интерфейсы
Почему так: По умолчанию MySQL слушает только localhost (127.0.0.1), что не позволяет подключаться с других серверов.

2.2 Перезапускаем MySQL
sudo systemctl restart mariadb   # или mysql

2.3 Проверяем, что порт слушается на всех интерфейсах
sudo ss -tulpn | grep 3306
Правильный вывод:
tcp   LISTEN   0   151   0.0.0.0:3306   0.0.0.0:*   users:(("mysqld",pid=...))

2.4 Настройка брандмауэра (если включён)
# Для ufw
sudo ufw allow from 192.168.0.90 to any port 3306
# Для firewalld
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="192.168.0.90" port protocol="tcp" port="3306" accept'
sudo firewall-cmd --reload

Этап 3: Остановка Zabbix Server и создание дампа БД

3.1 Останавливаем Zabbix Server на старом сервере
На Zabbix-сервере (192.168.0.90):
sudo systemctl stop zabbix-server
Почему это важно: Если не остановить сервер, в БД будут писаться новые данные во время дампа, что приведёт к рассинхронизации или коррупции данных.

3.2 Проверяем, что процесс действительно остановлен
sudo systemctl status zabbix-server

3.3 Создаём полный дамп БД
mysqldump -u root -p \
  --single-transaction \
  --routines \
  --triggers \
  --all-databases > /tmp/full_zabbix_backup.sql
  
Параметры с пояснениями:

Параметр	Зачем
--single-transaction	Создаёт согласованный снимок без блокировки таблиц (для InnoDB)
--routines	Сохраняет хранимые процедуры (важно для версий Zabbix 6.0+)
--triggers	Сохраняет триггеры — критически важно для работы Zabbix
--all-databases	Бэкапит все БД, а не только zabbix

Этап 4: Перенос данных на новый БД-сервер

4.1 Копируем дамп на новый сервер
scp /tmp/full_zabbix_backup.sql root@192.168.0.30:/tmp/
Если проблемы с правами, используем обычного пользователя - user@192.168.0.30:/tmp/

4.2 Создаём пользователя и БД на новом сервере
На новом БД-сервере (192.168.0.30) зайдите в MySQL:
sudo mysql -u root -p
Выполните SQL-команды:
-- Создаём БД
CREATE DATABASE zabbix CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;
-- Создаём пользователя для подключения с Zabbix-сервера
CREATE USER 'zabbix'@'192.168.0.90' IDENTIFIED BY 'ваш_безопасный_пароль';
-- Даём права
GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'192.168.0.90';
-- Если хотите временно разрешить с любого IP (для отладки)
-- CREATE USER 'zabbix'@'%' IDENTIFIED BY 'ваш_безопасный_пароль';
-- GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'%';
-- Применяем изменения
FLUSH PRIVILEGES;
-- Проверяем созданных пользователей
SELECT host, user FROM mysql.user WHERE user='zabbix';
EXIT;

4.3 Импортируем дамп
mysql -u root -p < /tmp/full_zabbix_backup.sql
Внимание: Импорт может занять от нескольких минут до часа в зависимости от объёма истории.

Этап 5: Настройка Zabbix Server для работы с новой БД
На Zabbix-сервере (192.168.0.90):

5.1 Редактируем конфиг Zabbix Server
sudo nano /etc/zabbix/zabbix_server.conf

Находим и меняем параметры:
# Было (БД на том же сервере)
# DBHost=localhost
DBName=zabbix
DBUser=zabbix
DBPassword=старый_пароль

# Стало (БД на отдельном сервере)
DBHost=192.168.0.30        # ← IP нового БД-сервера
DBName=zabbix
DBUser=zabbix
DBPassword=ваш_безопасный_пароль   # ← тот же пароль, что задали в MySQL
Важно: Если строка DBHost была закомментирована (#DBHost=localhost) или отсутствовала — убираем комментарий и указываем IP.

5.2 Редактируем конфиг веб-интерфейса
sudo nano /etc/zabbix/web/zabbix.conf.php
Меняем те же параметры:
<?php
$DB['TYPE']     = 'MYSQL';
$DB['SERVER']   = '192.168.0.30';   // ← меняем IP
$DB['PORT']     = '0';
$DB['DATABASE'] = 'zabbix';
$DB['USER']     = 'zabbix';
$DB['PASSWORD'] = 'ваш_безопасный_пароль';   // ← меняем пароль
// ... остальное без изменений
?>

5.3 Проверяем синтаксис PHP-файла
php -l /etc/zabbix/web/zabbix.conf.php
Должны увидеть:
No syntax errors detected in /etc/zabbix/web/zabbix.conf.php

Этап 6: Проверка подключения и запуск

6.1 Проверяем сетевую доступность порта БД
С Zabbix-сервера:
nc -zv 192.168.0.30 3306
Ожидаемый вывод:
Connection to 192.168.0.30 port 3306 [tcp/mysql] succeeded!

6.2 Проверяем подключение через MySQL-клиент
mysql -u zabbix -p -h 192.168.0.30 zabbix
Введите пароль
Если зашли в консоль MySQL — всё работает
Выход: EXIT;

6.3 Запускаем Zabbix Server
sudo systemctl start zabbix-server

6.4 Проверяем статус и логи
sudo systemctl status zabbix-server
sudo tail -f /var/log/zabbix/zabbix_server.log
Признаки успешной работы:
Нет строк с database error, connection refused или access denied
Появляются строки вида sending list of active checks to ...
Данные отображаются в веб-интерфейсе

Этап 7: Очистка и завершение

7.1 Убеждаемся, что всё работает
Проверьте в веб-интерфейсе:
Активны ли хосты
Появляются ли новые данные
Строятся ли графики

7.2 Останавливаем старую БД (опционально, через несколько дней наблюдения)
После того как убедились в стабильной работе:

# На старом сервере (192.168.0.90) останавливаем локальную БД
sudo systemctl stop mysql   # или mariadb
sudo systemctl disable mysql

7.3 Обновляем бэкап-стратегию
Теперь бэкапить нужно:
Новый БД-сервер — дампы БД
Zabbix-сервер — конфиги (/etc/zabbix/)

Ключевые моменты, которые нельзя пропускать
№	Момент	Почему важен
1	Остановка Zabbix Server перед дампом	Предотвращает потерю данных и рассинхронизацию
2	Использование --routines и --triggers в дампе	Без них Zabbix 6.0+ не будет работать корректно
3	bind-address = 0.0.0.0 на БД-сервере	Без этого MySQL не принимает внешние подключения
4	Создание пользователя 'zabbix'@'IP_ZABBIX_SERVER'	Без этого будет ошибка Host is not allowed to connect
5	Правка обоих конфигов (server.conf и zabbix.conf.php)	Если пропустить один — либо сервер, либо веб не будет работать
6	Проверка через nc и mysql CLI перед запуском	Позволяет отловить проблемы до запуска сервера
