1. Установка SNMP
bash
sudo apt update
sudo apt install -y snmpd
2. Настройка /etc/snmp/snmpd.conf
bash
sudo nano /etc/snmp/snmpd.conf
Минимальный рабочий конфиг (замените всё содержимое):

bash
# Где слушать
agentAddress udp:161

# Доступ
rocommunity public default --- не факт!!!

# Информация о системе (опционально)
sysLocation "Server Room"
sysContact "admin@example.com"
3. Перезапуск и проверка
bash
sudo systemctl restart snmpd
sudo systemctl enable snmpd

# Проверка локально
snmpwalk -v2c -c public 127.0.0.1 .1.3.6.1.2.1.1
4. Открыть порт в фаерволе (если включён)
bash
sudo ufw allow 161/udp
Готово!
Прокси 172.16.0.91 теперь сможет опрашивать сервер по SNMP.


НА ЗАБИКС СЕРВЕРЕ
Шаг 1: Добавление SNMP-интерфейса на хост
Вам нужно перейти в настройки хоста и явно добавить ему интерфейс типа SNMP.
В веб-интерфейсе Zabbix перейдите в Настройка (Configuration) → Хосты (Hosts).

Найдите в списке ваш хост и нажмите на него.
Перейдите в раздел Интерфейсы (Interfaces).
Нажмите на кнопку Добавить (Add) и выберите тип SNMP.
Заполните параметры:
IP-адрес/DNS: Введите IP-адрес вашего сервера eve-ng.
Порт: Оставьте стандартный 161 (или укажите ваш, если он изменён).
Версия SNMP: Выберите версию (например, SNMPv2 или SNMPv3). Обычно используется SNMPv2.
Сообщество (Community): Укажите строку сообщества (например, public). Она должна совпадать с той, что настроена на сервере eve-ng в его SNMP-сервисе (например, в snmpd.conf) .

Шаг 2: Проверка макроса сообщества (Community macro)
Шаблон "Linux by SNMP" использует макрос {$SNMP_COMMUNITY} для подстановки пароля (строки сообщества) во все элементы данных . Проще задать его один раз на уровне хоста, чем править каждый элемент.
В окне редактирования хоста eve-ng перейдите на вкладку Макросы (Macros).
Нажмите Добавить.
В поле Макрос введите: {$SNMP_COMMUNITY}
В поле Значение введите вашу строку сообщества (например, public).
После этого нажмите Обновить (Update) внизу формы, чтобы сохранить изменения хоста.

Шаг 3: Принудительное обновление (на случай, если ошибка осталась)
Иногда, даже после добавления интерфейса, элементы данных, унаследованные от шаблона, "не видят" его. Это известная проблема в некоторых версиях Zabbix . Если данные не пошли, можно попробовать:
"Потрясти" шаблон: Снять шаблон "Linux by SNMP" с хоста, нажать Обновить, затем снова прикрепить его.
Очистить кэш конфигурации: Перезапустите сервер Zabbix (команда systemctl restart zabbix-server). Это заставит его перечитать все конфигурации и соответствия интерфейсов.
