ITIL (Information Technology Infrastructure Library) — это библиотека лучших практик управления IT-услугами. Представьте ITIL как "правила хорошего тона" для IT-отдела.
В ITIL есть понятие управление уровнем услуг (Service Level Management). Это процесс, который отвечает за:
Договорённости между IT и бизнесом (SLA)
Измерение реального качества (SLI)
Постановку целей (SLO)
Зачем это нужно бизнесу:
Без SLA-отчетов	Со SLA-отчетами
"Сайт вчера вечером тормозил, айтишники ничего не делают"	"В апреле доступность сайта составила 99.8% при плане 99.9%. Причина: два инцидента 15 апреля с 20:00 до 20:30"
IT и бизнес говорят на разных языках	IT говорит на языке бизнеса (доступность, простой, компенсации)
Непонятно, хорошо работает IT или плохо	Есть цифры и отчёты



Термин	Расшифровка	Что это простыми словами	Пример
SLI	Service Level Indicator	Факт — что мы измеряем	"Сайт был доступен 99.5% времени в этом месяце"
SLO	Service Level Objective	Цель — как должно быть	"Обещаем доступность 99.9%"
SLA	Service Level Agreement	Договор — что будет, если цель не достигнута	"Если доступность упадет ниже 99.9%, вернем 10% стоимости услуги"


Чтобы построить отчет SLA в Zabbix, нужно пройти четыре шага, создавая цепочку: Триггер → Услуга → SLA.
Триггер на вашем хосте zabbiq-agent-pfsense генерирует проблему и вешает на неё тег.
Услуга ловит проблему по этому тегу и формирует показатель доступности (SLI).

SLA-правило подключается к услуге по её тегу и сверяет полученный SLI с вашей целью SLO (99%), после чего строит отчет.
Ниже я опишу создание каждого элемента на примере мониторинга вашего сервера MySQL.

Шаг 1. Настройка тега на триггере
Где: Configuration → Hosts → выберите ваш хост с MySQL → Triggers → выберите или создайте триггер (например, "MySQL is down").
Что делаем: Перейдите на вкладку Tags и нажмите Add.
Параметры тега:
Name (Имя): service
Value (Значение): mysql_db
Зачем: Когда этот триггер сработает, Zabbix "прометит" проблему тегом service:mysql_db . Наша будущая услуга будет "слушать" именно этот тег.

Шаг 2. Создание Услуги (Service)
Услуга — это логический объект, который мы хотим мониторить (например, "Сервер баз данных").
Переход: В веб-интерфейсе зайдите в Services → Services .
Режим редактирования: Нажмите кнопку Edit (Карандаш) в правом верхнем углу, затем Create service.
Основные параметры:
Name: Введите имя, например, MySQL Database Server.
Parent services: Оставьте пустым (это будет корневая услуга).
Проблемные теги (Problem tags): Здесь мы связываем услугу с триггерами.
Нажмите Add в блоке Problem tags.
Name: service (имя тега с триггера).
Value: mysql_db .
Теги сервиса (Tags): Эти теги понадобятся позже, чтобы связать услугу с SLA-правилом.
Нажмите Add в блоке Tags.
Name: SLA.
Value: MySQL_99 .
Сохраните услугу, нажав Add.

Шаг 3. Настройка SLA-правила
SLA-правило — это, по сути, шаблон для отчета. Вы уже создали такое правило на скриншоте, но теперь мы можем его правильно заполнить.
Переход: Services → SLA .
Создание/редактирование: Нажмите Create SLA (или выберите ваше правило MySLA для редактирования).
Заполните вкладку SLA:
Name: MySQL 99% SLA (хорошо бы давать понятные имена).
SLO: 99 (ваша цель в процентах) .
Reporting period: Выберите Monthly .
Timezone: Установите (UTC+03:00) Europe/Minsk.
Schedule: Оставьте 24x7.
Effective date: Оставьте сегодняшнюю или прошедшую дату.

Связываем SLA с услугой через Service tags:
Это самый важный момент. Нажмите Add в блоке Service tags.
Name: SLA (это имя тега услуги, который мы задали на шаге 2.5).
Operation: Equals.
Value: MySQL_99 (это значение тега услуги) .

Сохраните правило (убедитесь, что флажок Enabled активен).

⚙️ Как это работает (Алгоритм)
Проблема: На вашем pfSense хосте срабатывает триггер "MySQL is down".
Метка: Триггер вешает на проблему тег service:mysql_db.
Фильтрация: Услуга MySQL Database Server "видит" этот тег через свои Problem tags.
Расчет: Услуга начинает считать время недоступности.
Отчет: SLA-правило MySQL 99% SLA с помощью своего тега SLA:MySQL_99 обращается к услуге, забирает данные о времени недоступности и строит отчет, сравнивая их с вашей целью в 99% .
